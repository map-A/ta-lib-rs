## 2. How to Add a New Indicator (Full Flow)

### Step 1: Implement the core algorithm (`polars-ta-core`)

**File path**: `crates/polars-ta-core/src/{category}/{name}.rs`

- `category` is one of: `trend` / `oscillator` / `volume` / `volatility` / `statistic` / `math_ops` / `math_transform` / `price_transform`

**Function signature conventions**:

```rust
// Single input, single output (e.g. SMA, RSI)
pub fn {name}(data: &[f64], period: usize) -> Vec<f64>

// Multi-column input (e.g. ATR)
pub fn {name}(high: &[f64], low: &[f64], close: &[f64], period: usize) -> Vec<f64>

// Multi-output (e.g. MACD)
pub struct MacdOutput {
    pub macd:   Vec<f64>,
    pub signal: Vec<f64>,
    pub hist:   Vec<f64>,
}
pub fn macd(data: &[f64], fast: usize, slow: usize, signal: usize) -> MacdOutput
```

**Rules you must follow**:

1. **No panics**: `if data.len() < min_required { return vec![]; }` or return empty struct
2. **Output length** = `input.len() - lookback` (see ta-lib docs for lookback)
3. **NaN propagation**: NaN input → NaN in output windows that overlap it
4. **Zero external dependencies**: only `std`, no external crates
5. **Inline doc comments**: must include parameters, output length formula, code example

**Minimum template**:

```rust
/// {IndicatorName} — {description}
///
/// # Parameters
/// - `data`   — input price series
/// - `period` — window length
///
/// # Output
/// Length = `data.len() - {lookback_expr}`
/// Returns empty `Vec` when `data.len() < {min_input_len}`.
pub fn {name}(data: &[f64], period: usize) -> Vec<f64> {
    let n = data.len();
    let lookback = /* formula */;
    if period == 0 || n <= lookback {
        return vec![];
    }
    let mut out = Vec::with_capacity(n - lookback);
    // algorithm
    out
}
```


### Step 3: Implement the Polars Series wrapper (`polars-ta`)

**File path**: `crates/polars-ta/src/{category}/{name}.rs`

```rust
// Reference: crates/polars-ta/src/trend/sma.rs
use polars_core::prelude::*;
use polars_ta_core::{category}::{name} as {name}_core;

pub fn {name}_series(series: &Series, period: usize) -> PolarsResult<Series> {
    let ca = series.cast(&DataType::Float64)?;
    let ca = ca.f64()?;
    let data: Vec<f64> = ca.into_iter().map(|v| v.unwrap_or(f64::NAN)).collect();
    let result = {name}_core(&data, period);
    Ok(Series::new(series.name().clone(), result))
}
```

### Step 4: Generate golden files

Add a generator function to `scripts/generate_golden.py`:

```python
def generate_{name}(output_dir: Path, period: int = 20):
    """Generate {NAME} golden files."""
    print(f"\n[{NAME}] period={period}")
    lookback = period - 1  # replace with correct formula
    for dataset_name, make_fn in DATASETS.items():
        data = make_fn()
        close = data["close"]
        result = talib.{TALIB_NAME}(close, timeperiod=period)
        _write_golden(output_dir, "{name}", {"period": period},
                      {"close": close}, result, lookback, dataset_name)
    # boundary cases
    bdata = make_boundary_exact(lookback)
    result = talib.{TALIB_NAME}(bdata["close"], timeperiod=period)
    _write_golden(output_dir, "{name}", {"period": period},
                  {"close": bdata["close"]}, result, lookback, "boundary_exact")
    sdata = make_boundary_short(lookback)
    result = talib.{TALIB_NAME}(sdata["close"], timeperiod=period)
    _write_golden(output_dir, "{name}", {"period": period},
                  {"close": sdata["close"]}, result, lookback, "boundary_short")

GENERATORS["{name}"] = lambda out_dir: generate_{name}(out_dir)
```

Then run:

```bash
python scripts/generate_golden.py --indicator {name}
```

### Step 5: Write the golden test

**File path**: `crates/polars-ta-verify/tests/golden_{name}.rs`

```rust
use polars_ta_core::{category}::{name};
use polars_ta_verify::golden::{assert_close, load_golden_file};
use std::path::PathBuf;

fn golden_path(filename: &str) -> PathBuf {
    PathBuf::from(env!("CARGO_MANIFEST_DIR"))
        .parent().unwrap().parent().unwrap()
        .join("tests").join("golden").join(filename)
}

fn run_{name}_golden(filename: &str, period: usize, epsilon: f64) {
    let path = golden_path(filename);
    if !path.exists() { println!("SKIP: {filename}"); return; }
    let golden = load_golden_file(&path).unwrap();
    let input = golden.close_input().unwrap();
    let actual = {name}(&input, period);
    assert_close(&actual, &golden.output.values, epsilon,
                 &format!("{name}/period={period}"));
}

#[test] fn {name}_normal_1000()    { run_{name}_golden("{name}_period20_normal_1000.json", 20, 1e-10); }
#[test] fn {name}_boundary_exact() { run_{name}_golden("{name}_period20_boundary_exact.json", 20, 1e-10); }
#[test] fn {name}_with_nan()       { run_{name}_golden("{name}_period20_with_nan_5pct.json", 20, 1e-10); }
#[test] fn {name}_all_same_value() { run_{name}_golden("{name}_period20_all_same_value.json", 20, 1e-10); }
#[test] fn {name}_real_btcusdt()   { run_{name}_golden("{name}_period20_real_btcusdt_1d.json", 20, 1e-10); }
#[test] fn {name}_real_flat()      { run_{name}_golden("{name}_period20_real_flat_period.json", 20, 1e-10); }
```

| Fail | `> 1e-7` | Must be fixed before merging |

## 4. Golden Test Failure Debugging Flow

```
1. Note the failing index and deviation (assert_close prints both)
2. Compare against ta-lib source: https://github.com/TA-Lib/ta-lib/tree/master/src/ta_func
3. Common causes:
   a. EMA seeding — ta-lib seeds with SMA(data[0..period]), not the first value
   b. Wilder smoothing — RSI/ATR use alpha=1/period, not 2/(period+1)
   c. Wrong lookback — output length mismatch shifts all values
   d. Multi-column alignment — high/low/close must be same length and aligned
4. Fix and re-run: cargo test --package polars-ta-verify
5. If still failing after 3 attempts, relax to 1e-7 and document root cause in code
```

### 5b. Implementation notes

| Indicator | Key notes |
|-----------|-----------|
| EMA | Seed = SMA(data[0..period]); alpha = 2/(period+1) |
| RSI | Wilder smoothing (alpha = 1/period); lookback = period |
| ATR | Wilder smoothing; lookback = period; TrueRange = max(H-L, \|H-PC\|, \|L-PC\|) |
| MACD | macd = EMA(fast) − EMA(slow); signal = EMA(macd, signal_period) |
| BBands | middle = SMA; uses **population** std dev (not sample) |
| SAR | State machine; see ta-lib ta_SAR.c for reference |
| ADX | Based on ±DI/DX with Wilder smoothing; lookback = 2×period−1 |
| MACDFIX | Uses **fixed multipliers** k_fast=0.15, k_slow=0.075 — NOT 2/(period+1) |
| SAREXT | Negates output value when in **short position** (SAR does not) |
| Math Transform | All 15 functions are element-wise; lookback=0; output length = input length |
| Math Ops (MAX/MIN/SUM) | Rolling window using O(n) monotone deque / sliding sum |

## 6. Performance Checklist

- [ ] Use sliding window, not recompute-from-scratch (avoid O(n²))
- [ ] `Vec::with_capacity(n - lookback)` — pre-allocate
- [ ] No unnecessary `.clone()`
- [ ] No `String` or `Box` allocation in hot path
- [ ] Target: throughput ≥ 95% of ta-lib C (hard gate: ≥ 80%)

## 7. Multi-column indicator golden file generation

For indicators that need high/low/close (e.g. ATR, CCI):

```python
def generate_atr(output_dir: Path, period: int = 14):
    lookback = period
    for name, make_fn in DATASETS.items():
        data = make_fn()
        result = talib.ATR(data["high"], data["low"], data["close"], timeperiod=period)
        _write_golden(output_dir, "atr", {"period": period},
                      {"high": data["high"], "low": data["low"], "close": data["close"]},
                      result, lookback, name)
```

The golden file's `input` field contains multiple columns; read them with `golden.get_input("high")` on the Rust side.
